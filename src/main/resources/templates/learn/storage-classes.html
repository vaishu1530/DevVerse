Storage Classes in C

In C, storage classes define the scope, lifetime, and visibility of variables or functions. They tell the compiler how to store the variable and how long it should be active in memory.

1. Types of Storage Classes

Storage Class Scope Lifetime Default Value
auto Local Within block Garbage value
register Local Within block Garbage value
static Local/Global Entire program 0
extern Global Entire program 0

2. Example Code: Using static

#include <stdio.h> void counter() { static int count = 0; count++; printf("Count: %d\n", count); } int main() { counter(); counter(); counter(); return 0; }

Output:
Count: 1
Count: 2
Count: 3

Explanation: The variable count retains its value between function calls because it's declared as static.

3. Explanation of Each Class

  • auto: Default for local variables inside functions. Memory allocated on stack.
  • register: Stores variable in CPU register for faster access (hint to compiler).
  • static: Retains value across multiple function calls. Memory allocated only once.
  • extern: Declares a global variable defined in another file.

4. Tips

  • Use static to preserve variable values across function calls.
  • Use extern for accessing global variables across files.
